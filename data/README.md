# 원본 데이터 (5종 DB)

이 디렉터리(`data/raw/`)에 보고서 2장의 5종 DB를 업로드하세요. **저장소에 커밋되지 않습니다**
(`.gitignore` 처리 — 개인/대용량 데이터). 빌드 시 ETL이 읽어 `public/data/dataLayer.json`을 생성합니다.

| 파일명(기본값) | 내용 | 규모 |
|---|---|---|
| `5등급_9등급_환산_DB.xlsx` | 5→9 환산표 (0.05 간격) | 51행 |
| `2026_수시입결_DB.xlsx` | 수시 입결 (교과/종합/논술/실기) | 26,145행 |
| `대학교과전형_DB.xlsx` | 교과전형 상세 (수도권 31개교) | 31개교 |
| `2028_권역별대학별_권장과목DB.xlsx` | 종합전형 권장과목 (Phase 2) | 1,358행 |
| `2028_전공관련과목현황_DB.xlsx` | 전공관련 과목 (Phase 2) | 907행 |

## 업로드 후 절차
1. 실제 파일명이 위와 다르면 `scripts/etl/columnMap.ts`의 `FILES`를 수정.
2. 실제 컬럼 헤더에 맞춰 같은 파일의 `*_COLS` 매핑을 수정.
3. `npm run etl` 실행 → 콘솔에서 `입결 ↔ 교과전형 31/31 (100%)` 매칭률 확인.

> 데이터가 아직 없으면 `npm run etl:mock`으로 샘플 데이터를 생성해 앱을 구동할 수 있습니다.
